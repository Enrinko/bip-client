# Безопасность

> **Язык**: Русский · [English](SECURITY.md)

Контракт безопасности **bip-client** — JavaFX-клиента, который обращается к серверу
[bipApi](https://github.com/Enrinko/bipApi) по REST и читает/пишет XLSX через Apache POI. Цель
поставки — автономный **Windows `.exe`** (jpackage app-image), поэтому пункты baseline про
контейнеры/Kubernetes помечены N/A и заменены контролями уровня desktop и supply-chain.

## Как сообщить об уязвимости

**Не** открывайте публичный issue. Пишите на **enrinkopiece@gmail.com**: описание, шаги
воспроизведения, затронутый коммит. Дайте разумное окно на фикс до публичного раскрытия.

## Модель угроз (в скоупе)

- Вредоносный или подменённый релизный бинарь (supply-chain / поддельный `.exe`).
- Уязвимая сторонняя зависимость (POI, OkHttp и их транзитивные).
- Перехват/подмена трафика клиент↔сервер (открытый HTTP).
- Утечка локальных данных по нагрузке (XLSX) через систему контроля версий.

Не в скоупе: компрометация ОС пользователя, физический доступ к машине, атаки на сам сервер
(покрыты `SECURITY.md` серверного репозитория).

## Overrides (заявленные проектом)

| Пункт | Override | Причина | Рекомендация |
| --- | --- | --- | --- |
| Base URL сервера | По умолчанию открытый `http://localhost:28242/bipapi/` | Дефолт для локальной разработки; клиент и сервер на одной машине. | Задайте `BIP_API_BASE_URL` (или `-Dbip.api.base-url`) с **`https://`**, когда сервер удалённый — приложение уже учитывает override. |
| Trivy (#8) | `exit-code: 0` (только отчёт) | Новый пайплайн; не делать билд вечно красным до появления baseline-скана. | Разобрать первый скан, затем `exit-code: 1` для гейта. |
| Legacy `*.xlsx` в истории | Таблицы закоммичены до правила ignore | Могут содержать реальные данные по нагрузке (ФИО, группы). | `git rm --cached *.xlsx`; при чувствительности — очистить историю `git filter-repo`. |
| Пиннинг GitHub Actions | По тегам, не по commit SHA | Читаемость; как в серверном репозитории. | Пиннить по SHA после включения Dependabot. |

## Обращение с секретами и данными

| Слой | Механизм |
| --- | --- |
| URL сервера | `config.properties` / env `BIP_API_BASE_URL` / `-Dbip.api.base-url` — **не секрет** |
| Credentials | нет — клиент пока не отправляет аутентификацию |
| Локальные таблицы | остаются на диске пользователя; `*.xlsx` в gitignore |

Если сервер начнёт требовать аутентификацию — храните токены в хранилище учётных данных ОС
(Windows Credential Manager / Keychain), никогда в `config.properties` или исходниках.
